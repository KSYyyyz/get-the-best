# 《壮志凌云》V2-0.15 对象轮廓反馈与预设办公室验收报告

## 本轮目标

本轮根据实机截图反馈，修正员工和设施的对象级交互表现：

- 员工不再用地面黄色底座表达选中。
- 员工拖拽不再复用区域选择格子预览。
- 鼠标滑过、选中、拖拽时，对象以实例轮廓描边表达状态。
- 员工拖拽放下后清除对象选中和区域预览残留。
- 设施补齐对象实例拖拽逻辑，和员工保持一致的交互原则。
- 沙盒默认预设一个更完整的办公室，便于后续测试员工行动、设施和房间关系。

## 根因

原交互把员工和设施当作“格子上的内容”处理：

- 员工选中由 `Employee3DRenderer` 在地面绘制黄色底座。
- 员工拖拽用 `PlacementPreview3DController.ShowSelectionRect` 表达目标格。
- 点击命中先把屏幕点投到地面格，再用格子找员工或设施。

在 2.5D 视角下，玩家点击员工身体时，屏幕点投到地面后可能落在员工身后的格子，导致没有命中员工，反而进入区域选择状态。这就是拖拽结束后出现绿色选区残留的直接原因。

## 已完成

1. 员工改为对象轮廓反馈。
   - 移除了黄色地面底座。
   - 新增员工实例轮廓描边。
   - 鼠标滑过、选中、拖拽时保留轮廓。
   - 拖拽非法目标时员工本体变红，不再用大面积格子反馈。

2. 员工拖拽结束清理残留。
   - 合法放下后刷新员工位置。
   - 合法或非法放下后都清除拖拽预览、对象选中和区域预览。
   - 避免拖拽结束后留下绿色选区或黄色底座。

3. 设施补齐对象级交互。
   - 设施支持鼠标滑过轮廓描边。
   - 设施支持选中轮廓描边。
   - 设施支持按住拖拽移动。
   - 设施拖拽合法性继续沿用当前房间类型和占用格约束，不加入经营规则。

4. 命中逻辑改为屏幕投影对象命中。
   - 新增员工和设施的屏幕投影命中判断。
   - 鼠标点击对象本体时不再依赖地面格子命中。
   - 区域选择只在没有命中对象时启动。

5. 预设办公室沙盒。
   - 默认生成研发室、市场室、服务器室。
   - 默认生成门、办公桌、产品白板、服务器机柜。
   - 默认员工站在研发室中，便于后续测试移动、设施占用和导航。

## 实机验证

使用 Godot MCP 运行 `res://scenes/main.tscn`。

验证结果：

- 默认画面可看到三个房间、门、设施和员工。
- 员工从研发室内拖拽到相邻格后，员工位置更新，未留下绿色区域选区。
- 员工拖拽放下后未保留黄色底座或对象选中状态。
- 产品白板设施可作为实例拖拽移动。
- `get_errors(include_warnings=false)` 返回 0 error。

## 后续建议

下一步可以继续回到 V2-0.15 原本的“空间导航基础”：

- 以当前预设办公室为测试场景。
- 建立可行走格、阻挡格、门通行和设施占用的统一判断。
- 让员工拖拽、后续自动移动和设施交互共用同一套空间通行判断。
