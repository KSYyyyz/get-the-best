# 《壮志凌云》V2-0.18 平滑移动、精确命中与墙门表现验收报告

## 本轮目标

修正办公室沙盒里四类直接影响手感和空间可信度的问题：

1. 员工和设施拿起移动时不再一格一格硬跳。
2. 员工和设施不能在离实例很远的位置被误选中。
3. 办公室外墙和房间墙体去掉黑色格线，改成更像真实墙面的浅色实体墙。
4. 门改成贴墙的竖向门板和门框，墙体整体加高，增强 3D 体积感。

## 已完成

1. 平滑移动
   - `Employee3DRenderer` 和 `Facility3DRenderer` 在刷新前记录上一帧渲染位置。
   - 拖拽目标格变化时，通过 `CreateTween()` 在 0.12 秒内补间到新目标位置。
   - 设施节点新增稳定命名 `Facility_{id}`，与员工节点 `Employee_{id}` 保持一致。

2. 精确命中
   - 移除统一的 72px 命中半径。
   - 员工命中半径收紧为 28px。
   - 设施命中半径收紧为 34px。
   - 拾取仍基于实例投影点，而不是回退到粗糙的格子选择。

3. 墙体表现
   - 房间墙和办公室外墙不再使用带格线的 prototype wall texture。
   - 墙体改用浅色实体材质，并保留顶部 trim。
   - 房间墙和外墙高度从 `0.82 * GridSize` 提升到 `1.16 * GridSize`。

4. 门表现
   - 从 Kenney `Building Kit` 引入墙和门相关资源，并登记到资产索引。
   - 可见门改成贴墙的竖向门板和门框，不再是低矮地面块。
   - 门高度提升到 `0.92 * GridSize`，位置改为按竖向门板中心对齐。

## 实机验证

使用 Godot MCP 运行 `res://scenes/main.tscn`：

- 默认预设办公室可以正常显示加高浅色墙体和贴墙竖向门板。
- 点击员工后进入“点击放下”拿起状态，员工模型仍保持实例显示。
- 鼠标移动到新格时，员工渲染节点会补间到新位置。
- `Employee_1`、`Employee_2`、`Employee_3` 运行时节点名稳定。
- Godot 错误面板：0 error。

## 自动化验证

新增测试覆盖：

- 员工与设施拖拽渲染器必须包含平滑移动补间。
- 员工与设施必须使用更小的实例投影命中半径。
- 墙和门相关 Kenney Building Kit 资源必须登记在资产索引。
- 房间墙、外墙和门尺寸必须使用本轮提升后的高度参数。
- 房间墙和外墙不再引用带格线的旧墙面纹理。
