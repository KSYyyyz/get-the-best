# 《壮志凌云》V2-0.13 员工拖拽与默认镜头验收报告

## 本轮目标

本轮围绕“员工可被直接拖动”和“打开游戏后镜头不再落到角落”做交互基线修正。

## 已完成

1. 员工现在支持在普通指针状态下拖动。
   - 鼠标按住员工进入拖拽状态。
   - 拖到合法空格时显示绿色单格预览。
   - 拖到非法位置时显示红色单格预览。
   - 松手在合法格会移动员工。
   - 松手在非法格会保留原位置。

2. 员工位置变更保存在 `EmployeeStore`。
   - 新增 `CanMoveEmployee`。
   - 新增 `TryMoveEmployee`。
   - 当前合法性先限定为办公室范围内且没有其他员工占用。
   - 后续房间、岗位、工位规则应由规则核心或专门交互服务继续接入，不在 UI 中复制经营规则。

3. 默认镜头修正为正视主操作区域。
   - 镜头默认 yaw 调整为正视方向。
   - 默认焦点落在员工所在的开局操作区。
   - 去掉启动时按全沙盒尺寸强制适配的行为。
   - 修复了启动后边缘移动因为默认鼠标位置为 `(0, 0)` 而自动把镜头推到角落的问题。
   - 滚轮缩放步进从 `1.14` 调整为 `1.08`，让缩放更接近经营游戏的细腻手感。

## 和《疯狂游戏大亨2》的对比结论

当前最需要学习的是“开局镜头先服务经营对象，而不是展示整张空地图”。《疯狂游戏大亨2》的镜头会让玩家第一眼看到房间、员工和可操作热点；本轮把默认镜头从角落/全图展示改为居中正视员工区域，是朝这个方向收敛。

但镜头手感仍只是基础版：

- 还没有惯性或平滑过渡。
- 缩放没有围绕鼠标指向点做焦点保持。
- 场景缺少真实办公室房间与设施后，镜头舒适度只能部分验证。

这些应放到后续“镜头手感专项”中继续做，而不是在员工拖拽这一轮里扩散范围。

## 实机验证

使用 Godot MCP 运行 `res://scenes/main.tscn`。

验证结果：

- 默认镜头不再漂移到左上角。
- 员工可从原格拖到相邻空格。
- 拖动后运行时员工节点位置从 `x=-5,z=-45` 更新到 `x=-15,z=-45`。
- Godot 错误面板：0 error。

## 后续建议

下一步应进入“员工与设施放置约束”：

- 员工只能放到可站立地面。
- 员工应优先吸附到房间、工位、设施附近。
- 员工与设施的选择面板要展示可执行动作。
- 拖拽时应显示目标格的业务含义，例如“研发室空位”“无可用工位”“需要先建门”。
