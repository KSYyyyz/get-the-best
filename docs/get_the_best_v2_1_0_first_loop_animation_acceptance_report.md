# V2-1.0 第一局目标链路与基础动画验收报告

日期：2026-05-22

## 本轮目标

本轮开始从 V2-0 空间沙盒进入 V2-1 第一局闭环。玩家需要能理解当前阶段、下一目标、最近 Core 事件，以及员工为什么会去设施工作；同时建立基础动画状态层，为后续替换真实角色动作素材打底。

## 已完成内容

1. 第一局目标 HUD 基线
   - 现有经营反馈面板增加当前目标、下一步、最近 Core 事件。
   - HUD 信息从 `CoreOfficeSimulationResult` 派生。
   - Godot 侧不计算经营规则，只消费 Core 输出并负责表现。

2. 员工表现动画状态层
   - 新增 `EmployeePresentationAnimationState`。
   - 支持 `Idle`、`Walking`、`SittingDown`、`WorkingAtDesk`、`UsingStandingFacility`、`LeavingFacility`。
   - `TypingHands` 和程序化动作挂到状态层，后续可替换为 GLB/`AnimationPlayer`。

3. Core 意图到表现映射
   - `WorkAtDesk` 映射到办公桌工作表现。
   - 白板、服务器等非办公桌设施进入站立使用表现。
   - 普通移动结束后清理表现状态，避免残留工作或使用设施动画。

## 验收证据

- 脚手架测试覆盖第一局目标 HUD 与员工表现状态映射。
- Godot C# 构建通过。
- MCP 实机需确认：
  - 当前目标与下一步在经营反馈面板中可见。
  - 最近 Core 事件在经营反馈面板中可见。
  - 员工能平滑移动到办公桌并进入办公桌工作表现。
  - 至少一种非办公桌设施能触发站立使用表现。
  - `get_errors` 为 0。

## 后续限制

当前仍是程序化基础表现，不是最终角色动画质量。下一阶段需要接入真实角色动作素材，包括坐下/站起、走路、开门、打字，以及不同设施使用动作。
