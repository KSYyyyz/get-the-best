# V2-1.5 删除优先级与时间快捷键验收报告

日期：2026-05-23

## 本轮目标

修正办公室沙盒里两个高频操作问题：

1. 删除模式选中设施时，不能连同下方区域一起出售。
2. 时间流速需要支持键盘快捷键，减少测试员工行动和经营循环时的操作成本。

## 已完成内容

1. 删除优先级
   - `OfficeSelection3DController` 在删除框选结束后先出售选区内设施。
   - 如果本次删除命中了设施，会立即刷新设施显示并结束操作，不再继续删除区域。
   - `BuildModeController` 增加 `HasFacilitiesInSelection()`，区域删除前会检查选区内是否仍有设施。
   - `TryDeleteRoomsInSelection()` 和 `TryDeleteRoomAtCell()` 不再自动出售设施，避免规则入口绕过 UI 优先级。
   - 玩家要出售区域时，需要先把该区域格子上的设施出售或移动走。

2. 时间快捷键
   - `Space` 切换暂停/恢复。
   - `Tab` 在 `1x -> 2x -> 3x -> 1x` 之间循环。
   - 暂停后会记住上一次非暂停倍率，按 `Space` 恢复到暂停前的速度。

## 验收重点

- 删除设施是对象操作，删除区域是空间操作，二者不能在一次选区操作里混在一起执行。
- 区域删除入口本身也必须防御设施占用，防止后续新增按钮或快捷键时复现同类问题。
- 时间快捷键只控制 Godot 前端表现层的模拟推进，不改变 C# Core 经营规则。

## 后续限制

当前 `Tab` 是纯倍率循环，还没有绑定正式经营日历、自动暂停策略或月报节点。后续进入完整第一局闭环时，可以把它接入日期条、月报触发和关键事件自动暂停。
