# 《壮志凌云》V2-0.16 实例描边与点击拿起拖拽验收报告

## 本轮目标

根据实机截图反馈，修正上一轮“对象轮廓”仍然像调试框的问题，并调整拖拽手感：

- 员工和设施的反馈必须是贴合实例本体的描边，而不是矩形框、黄色底座或地面选区。
- 鼠标滑过对象时显示描边。
- 对象被选中、拿起或拖拽移动时保持描边。
- 员工和设施拖拽改为“点击实例拿起、移动鼠标、再次点击放下”。
- 松开鼠标不再触发放下，避免按住式拖拽误触和选区残留。

## 已完成

1. 员工描边改为实例轮廓壳。
   - `Employee3DRenderer` 不再生成四根立柱和顶部横杆。
   - 描边改为稍微放大的员工身体、头部和腿部轮廓壳。
   - 描边材质启用发光和 grow，使效果更接近参考图中的物体边缘高亮。

2. 设施描边改为实例轮廓壳。
   - `Facility3DRenderer` 不再绘制设施上方矩形框。
   - 办公桌、产品白板、服务器机柜会先绘制一层放大的轮廓壳，再绘制设施本体。
   - 鼠标滑过、选中和拖拽时统一保持同一套对象级描边。

3. 拖拽方式改为点击拿起、点击放下。
   - 第一次左键点击员工或设施时进入拿起状态。
   - 移动鼠标时对象实例跟随目标格移动。
   - 第二次左键点击时尝试放下。
   - 单纯松开鼠标不会放下对象。
   - 合法目标提示“点击放下”，非法目标提示“不能放置”。

4. 放下后的状态清理保持不变。
   - 合法放下后清除拖拽预览、对象描边和地面预览。
   - 非法放下后对象返回原位，并清除拖拽预览和选中状态。
   - 不再出现员工放下后残留绿色选区或黄色底座的问题。

## 测试覆盖

新增和更新的结构测试覆盖：

- 员工不再使用 `AddOutlinePost`、`OutlineHeight`、`OutlineRadius` 等框架式轮廓。
- 设施不再使用 `HighlightStrokeSize` 顶部矩形框。
- 员工和设施渲染器都必须存在 `Add*OutlineShell` 和 `CreateOutlineMaterial`。
- 左键按下时负责拿起和放下，左键释放时不再调用员工或设施放下逻辑。

## 验收标准

- 鼠标滑过员工和设施时显示贴合实例的蓝色描边。
- 点击员工或设施后对象保持描边并进入拿起状态。
- 移动鼠标时对象跟随目标格移动。
- 再次点击合法格后对象落位，并清除拖拽/选中反馈。
- 再次点击非法格后对象回到原位，并清除拖拽/选中反馈。
- Godot MCP 实机运行无 error。
