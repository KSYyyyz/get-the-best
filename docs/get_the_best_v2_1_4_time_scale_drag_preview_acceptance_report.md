# V2-1.4 时间流速与工作员工拖拽预览验收报告

日期：2026-05-23

## 本轮目标

玩家需要能调整办公室沙盒中的时间流速，并且在员工处于工作中时仍能被正常拿起移动。此前工作员工被拖拽时，预览不会离开设施座位，看起来像模型仍然留在工位上。

## 已完成内容

1. 时间流速控制
   - 新增 `TimeScalePanel`，提供暂停、1x、2x、3x 按钮。
   - 新增 `TimeScaleHudController`，只控制 Godot 表现层的时间推进。
   - `EmployeeAutonomyController` 增加 `_simulationTimeScale`，自主行动 tick 和 Core tick 的前端触发节奏会按倍率调整。
   - `Employee3DRenderer` 增加表现层倍率，员工路径移动和循环动画会跟随暂停/倍速。

2. 工作员工拖拽预览修复
   - 拿起员工时立即调用 `ClearEmployeePresentationState()`，释放工作状态和设施占用表现。
   - 员工 renderer 在拖拽预览期间不再使用 `EmployeeWorkPose` 覆盖目标位置。
   - 工作中的员工被点击拿起后，会显示移动预览，而不是继续贴在办公桌座位上。

## 验收重点

- 暂停/倍速只属于 Godot 前端表现和 tick 触发节奏，不改变 C# Core 的经营规则。
- 鼠标和摄像机操作不使用全局 `Engine.TimeScale`，避免 2x/3x 时镜头手感一起变化。
- 员工从工作设施上拿起后，会先中断当前前端工作表现；后续是否回到设施工作仍由自主行动和 Core 意图继续决定。

## 后续限制

当前时间控制是沙盒级按钮，还没有快捷键，也没有把月/日时间轴做成正式经营日历。后续进入更完整的第一局闭环时，可以把它升级为日历条、当前日期、月报触发点和自动暂停选项。
