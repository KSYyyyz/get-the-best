# V2-0.4 设施物件表现与对象上下文基线验收报告

状态：已完成
日期：2026-05-20
适用项目：《壮志凌云 / Get The Best》

## 1. 本轮目标

V2-0.4 的目标是把 V2-0.3 的纯色设施占位推进到“可替换物件表现”阶段。

本轮只处理设施表现层、设施定义、第三方占位素材登记和基础选择反馈，不进入员工系统，不计算经营收益、成本、现金、收入或用户。

## 2. 已完成内容

- 从文档中已确定的公开素材库策略出发，优先选用 Kenney CC0 素材。
- 使用 Kenney `Furniture Kit` 的等距 PNG 渲染作为临时占位素材。
- 新增第三方占位素材清单：
  - `godot/GetTheBestGodot/assets/third_party_placeholder_assets/asset-index.json`
- 新增设施定义目录：
  - `godot/GetTheBestGodot/scripts/FacilityDefinitionCatalog.cs`
- 每类设施现在拥有基础表现定义：
  - 名称
  - 所属房间类型
  - 占地尺寸，当前均为 `1x1`
  - 是否是员工工位
  - 纹理路径
- `FacilityRenderer` 从设施定义中加载 `Texture2D`，优先绘制纹理，加载失败时保留色块兜底。

## 4. 明确未做

- 未接入员工寻路、坐工位或工作状态。
- 未加入设施经营收益或维护成本。
- 未把设施规则写入 Godot UI。
- 未引入插件。
- 未引入 itch.io、OpenGameArt 或 Freesound 资源。
- 未制作最终商业美术。

## 5. 验收标准

- `asset-index.json` 中登记所有第三方占位素材的来源、许可证、当前用途和替换计划。
- 设施渲染不再只依赖纯色块。
- 三类设施仍遵守 V2-0.3 的放置房间规则：
  - 办公桌：研发室
  - 产品白板：市场室
  - 服务器机柜：服务器室
- 设施纹理加载失败时有可见兜底表现。
- 本轮不改变 C# Core 规则边界。

V2-0.5 应先让员工能够绑定办公桌、走到工位、显示工作状态，再考虑更复杂的疲劳、技能、情绪和月结产出。
