# 《壮志凌云 / Get The Best》V2 同类游戏研究执行文档

状态：V2 研究基线
日期：2026-05-20
范围：同类经营模拟游戏的核心循环、界面结构、阶段规划和反照搬边界

## 1. 研究目标

本文件不是资料摘抄，而是为《壮志凌云 / Get The Best》V2 建立执行方向。

研究目标：

- 明确“创业版疯狂游戏大亨2全量系统”的长期目标。
- 把长期目标拆成可执行阶段，避免一开始做成混乱面板游戏。
- 提炼同类游戏的可学习结构，而不是复制主题、素材、文本、数值或专有系统。
- 为 V2 工程架构、插件策略和首个可玩闭环提供依据。

## 2. 参考游戏结论概览

| 游戏 | 最值得学习 | 最大警告 |
| --- | --- | --- |
| 《疯狂游戏大亨2》 | 办公室空间、房间、员工、设施、市场、发行、主机和长期公司成长一体化 | 不要一开始追全量复杂度 |
| 《游戏发展国》 | 短循环、员工训练、阶段反馈、评分和销量带来的清晰成败节奏 | 不要把深度压缩成纯数值点击 |

| Software Inc. | 员工、团队、办公室、产品、支持、服务器和市场的深度模拟 | 不要过早复制其复杂管理负担 |

| STONKS-9800 | 数据、Mod、本地化、轻量模拟和长期可扩展内容结构 | 不要让数据层替代核心玩法 |

## 3. 《疯狂游戏大亨2》的关键启发

公开资料显示，《疯狂游戏大亨2》强调从小车库成长为大型游戏公司，玩家可以建设办公室、生产设施和服务器房，雇佣人才，制作游戏、主机和引擎，经营发行、市场分析和竞争公司。

对 V2 的启发：

- 办公室必须是主棋盘，不是仪表盘背景。
- 房间必须是功能模块，不是颜色区块。
- 员工、设施、房间、项目和市场必须形成同一条成长线。
- 公司阶段应该改变玩家关注点：早期做产品，中期做商业化，后期做组织和市场控制。
- 视觉上要让玩家一眼看到“这里是一家公司在运转”。

不能照搬：

- 不复制游戏开发题材的具体房间、IP、平台和发行细节。
- 不复制专有 UI、文本、数值、素材或事件。
- 不把目标变成游戏开发公司模拟，而是创业公司经营模拟。

转化为《壮志凌云 / Get The Best》：

- 游戏项目开发 -> SaaS/创业产品迭代。
- 游戏销量 -> 用户增长、MRR、客户质量、市场信任。
- 游戏引擎 -> 技术基础设施和产品能力。
- 发行/主机 -> 渠道、企业客户、平台化、生态位。
- 游戏公司办公室 -> 创业公司办公室。

## 4. 《游戏发展国》的关键启发

《游戏发展国》的优势不是系统大，而是节奏轻、反馈快。玩家通过员工、训练、项目、广告、评分和销量理解每一步经营结果。

对 V2 的启发：

- 第一局必须短，20-30 分钟要能经历完整起承转合。
- 每个阶段只引入少量概念。
- 员工成长要能直接映射到经营结果。
- 月度或项目反馈必须告诉玩家：刚才的选择为什么有效或失败。

不能照搬：

- 不把创业经营压成纯菜单选择。
- 不把办公室弱化为背景动画。
- 不复制评分/销量结构，而是设计适合创业公司的用户、收入、留存和现金流反馈。

转化为 V2：

- MVP 开发阶段：研发能力决定产品进度和稳定性。
- 商业验证阶段：销售和客户反馈决定首批收入。
- 复盘阶段：月报解释产品、用户、收入、成本和风险。

## 5. Game Dev Tycoon 的关键启发

Game Dev Tycoon 通过办公室阶段升级、开发阶段选择、市场反馈、评论和销量建立轻量但完整的经营循环。

对 V2 的启发：

- 初始复杂度要低，逐步解锁更深系统。
- 玩家每次扩张办公室或团队，都应该感到公司进入新阶段。
- 反馈要有戏剧性，但必须来自真实结算事实。

不能照搬：

- 不恢复 Web 前端路线。
- 不把所有决策做成弹窗向导。
- 不让 UI 压过办公室主体。

转化为 V2：

- 办公室扩张对应公司阶段。
- 阶段目标对应创业里程碑。
- 结局复盘对应公司故事。

## 6. Software Inc. 的关键启发

Software Inc. 更接近深度公司模拟。它把员工技能、团队、办公室、产品开发、支持、营销、服务器和长期市场竞争放在同一系统里。

对 V2 的启发：

- 员工不是简单加成，而是能力、薪资、需求、情绪、团队适配和风险的综合体。
- 房间布局、设施质量和员工需求会影响公司产出。
- 支持、服务器和客户满意度是产品商业化后的必要系统。

不能照搬：

- 不在 V2-1 就引入复杂排班、详细团队管理和完整产品线。
- 不让玩家陷入维护所有微观需求的压力。
- 不追求公式深度优先，而要先保证因果可读。

转化为 V2：

- 早期员工只需要岗位、技能、薪资、疲劳、情绪、成长、适配。
- 员工需求优先作为效率和风险信号，而不是日常照料模拟。
- 客服、服务器、销售等商业化系统分阶段加入。

## 7. Big Ambitions 的关键启发

Big Ambitions 的启发在于空间布置、业务扩张和员工运营形成可视化商业系统。

对 V2 的启发：

- 空间和设施要让玩家感到“这家公司是我布置出来的”。
- 员工行动和设施使用要反馈公司运转状态。
- 扩张应带来新机会和新成本。

不能照搬：

- 不把创业公司做成店铺经营。
- 不让采购、库存、店铺物流成为主循环。

转化为 V2：

- 办公室布局影响团队协作、瓶颈和能力。
- 扩张房间带来能力，同时增加固定成本和组织复杂度。

## 8. STONKS-9800 的关键启发

STONKS-9800 对 V2 的启发主要是工程组织：数据、本地化、Mod 和可扩展内容边界清晰。

对 V2 的启发：

- 内容数据和运行时代码要分离。
- 房间、设施、员工、事件、客户、竞争者和文案应尽早有数据定义。
- 未来可以为内容包、本地化和 Mod 留出结构，但不提前承诺公开 Mod 系统。

不能照搬：

- 不把轻量股票模拟做成 Startup Sim 的核心。
- 不让数据文件变成规则核心的替代品。

## 9. 《壮志凌云 / Get The Best》的长期目标

长期目标是做出“创业版疯狂游戏大亨2全量系统”，但实现必须分阶段。

长期系统包括：

- 办公室空间经营。
- 房间和设施体系。
- 员工岗位、技能、成长、需求和团队适配。
- 产品研发、商业化、用户增长、收入和客户成功。
- 竞争公司、市场趋势、董事会、融资和重大客户。
- 多阶段办公室扩张、多产品线、组织效率和中后期战略。
- 多结局复盘，包括破产、被收购、独角兽、IPO、创始人出局等。

## 10. 阶段性实现原则

### 第一阶段：不要追全量

V2-1 只追求一局 20-30 分钟可完成的闭环：

1. 建研发区。
2. 放研发设施。
3. 雇研发员工。
4. 做出 MVP。
5. 建销售区。
6. 雇销售员工。
7. 获得首批用户或收入。
8. 看结构化月报。
9. 进入阶段胜利或失败复盘。

### 第二阶段：房间扩展

房间逐步扩展为：

- 研发区：产品进度、质量、稳定性。
- 销售区：线索、客户转化、首批收入。
- 客服区：留存、满意度、投诉风险。
- 服务器区：用户容量、稳定性、运维成本。
- 会议区：融资、董事会、重大客户。
- 休息区：疲劳和情绪恢复。
- 厕所：基础需求和效率风险。
- 招聘区：候选人质量和招聘效率。

### 第三阶段：员工扩展

员工逐步扩展为：

- 岗位。
- 技能。
- 薪资。
- 特质。
- 疲劳。
- 情绪。
- 健康。
- 成长潜力。
- 岗位适配。
- 团队协作。

### 第四阶段：外部世界

外部世界逐步扩展为：

- 客户反馈。
- 竞争公司。
- 市场趋势。
- 融资节点。
- 董事会压力。
- 大客户机会。
- 媒体和声誉。

## 11. 反照搬清单

后续任何设计引用参考游戏时，必须同时写出“不照搬点”。

禁止：

- 复制参考游戏素材、文本、UI、数值、事件或专有名称。
- 以参考游戏的主题替代创业主题。
- 用“大而全”掩盖第一阶段闭环不清晰。
- 为了像《疯狂游戏大亨2》而牺牲 Startup Sim 已经形成的创业规则核心。

允许：

- 学习同类游戏如何组织办公室、房间、员工、设施和反馈。
- 学习阶段解锁方式。
- 学习玩家如何理解经营因果。
- 学习失败复盘如何形成故事。
