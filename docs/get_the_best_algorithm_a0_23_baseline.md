# 《壮志凌云 / Get The Best》Algorithm A0.23 经营算法基线

## 基线目标

A0.23 的目标是在 A0.22 员工行为与经营 tick 骨架之上，补齐第一批可接入的 Core 状态闭环。

本基线只覆盖四项最高优先级：

1. A0.23.1：Core 状态推进规则。
2. A0.23.2：Godot 到 Core 的事实快照桥接契约。
3. A0.23.3：员工行为生命周期规则。
4. A0.23.4：休息与恢复规则。

## 质量门槛

每项算法完成前必须满足：

- 先写 Core 测试并看到预期红灯。
- 实现后 Core 测试通过。
- 不引用 Godot API。
- 规则结果确定，不使用随机数。
- 补充独立中文验收文档。
- 全量本地验证通过后才允许提交和推送。

## 非目标

- 不做 AI 玩法。
- 不做完整商业化系统。
- 不做 UI 或美术表现层。
- 不把现金、疲劳、满意度、产能、项目进度等经营公式写进 Godot UI。
