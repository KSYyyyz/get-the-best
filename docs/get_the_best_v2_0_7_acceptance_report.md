# V2-0.7 相机与首屏构图基线验收报告

## 范围

本轮只处理 Godot 前端的办公室镜头与首屏观感，不接入经营规则。

目标是让玩家打开游戏后先看到一个有边界、有纵深的办公室沙盒，而不是铺满屏幕的平面网格编辑器。

## 已完成

- 默认正交镜头改为 2.5D 俯视构图，默认尺寸为 58，优先展示办公室空间边界。
- 滚轮缩放范围调整为 10 到 96，缩放步进更细。
- WASD 平移改为按当前镜头方向移动。
- 鼠标中键或右键拖拽改为按当前镜头方向平移。
- Q / E 支持围绕办公室焦点向左 / 向右旋转摄像机。
- 窗口尺寸变化时会重新适配默认视野，避免只适配 1280x720。
- 网格线透明度降低，并加入 5 格一条的主网格线，减少全屏表格感。
- 办公室边界增加角柱，强化“空间边界”而不是“无限网格”的第一印象。

## MCP 验收记录

- 使用项目本地 Godot MCP 桥接运行 `res://scenes/main.tscn`。
- 初始截图确认办公室边界、角柱和弱化网格可见。
- 发送 Q 键后，运行时 `OfficeCamera.rotation_degrees.y` 从默认约 35 变化到约 130，确认左旋有效。
- 发送 E 键后，运行时 `OfficeCamera.rotation_degrees.y` 回落到约 60，确认右旋有效。
- 发送滚轮输入后，运行时 `OfficeCamera.size` 从 58 变为约 50.9，确认缩放有效。
- Godot 错误面板检查为 0 error。

## 暂不处理

- 房间、设施、员工的最终 3D 美术替换不在本轮范围。
- 镜头角度预设、迷你地图、楼层切换不在本轮范围。
- 经营规则、月结、收入链路仍等待后续 C# Core 阶段。
