# 《壮志凌云》V2-0.14 员工拖拽模型预览验收报告

## 本轮目标

修复员工拖拽时“只有格子反馈，员工模型不跟手”的体验问题。

## 根因

V2-0.13 只把拖拽目标格传给了 `PlacementPreview3DController`，因此玩家只能看到绿色或红色格子。`Employee3DRenderer` 没有拖拽预览状态，所以被拖员工仍然绘制在原始存储位置，看起来像没有真正拖动员工。

## 已完成

1. `Employee3DRenderer` 新增员工拖拽预览状态。
   - `ShowEmployeeDragPreview`
   - `ClearEmployeeDragPreview`
   - `_dragPreviewEmployeeId`
   - `_dragPreviewCell`
   - `_dragPreviewIsLegal`

2. 拖拽时员工模型会临时渲染到目标格。
   - 合法目标：保留员工原色。
   - 非法目标：员工身体显示红色。
   - 模型略微抬高，强化“正在拖拽”的视觉状态。

3. 松手后清除预览状态。
   - 合法落点会更新员工真实格子。
   - 非法落点会刷新回原位置。

## 实机验证

使用 Godot MCP 运行 `res://scenes/main.tscn`。

验证结果：

- 按住员工并拖动时，员工模型跟随目标格移动。
- 目标格同时显示绿色合法预览。
- 松手后员工真实运行时位置更新到新格。
- 运行时节点位置从 `x=-5,z=-45` 移动到 `x=-15,z=-45`。
- Godot 错误面板：0 error。

## 员工系统方向修正

后续不应把员工系统局限为“拖到工位”。完整员工系统需要分层推进：

1. 员工实体层：属性、岗位、状态、需求、当前位置、目标位置。
2. 空间导航层：可行走地块、门、走廊、房间入口、设施占用。
3. 行为调度层：工作、休息、上厕所、吃饭、开会、培训、处理事件。
4. 房间服务层：房间提供工作位、服务容量、舒适度、噪音、动线成本。
5. 设施交互层：设施不是装饰，而是员工执行行为的目标点。
6. 经营反馈层：员工行为影响产能、效率、满意度、疲劳和流失风险。

也就是说，“和房间、工位、设施产生真实关系”只是员工系统的一部分，不能成为员工系统的边界。
