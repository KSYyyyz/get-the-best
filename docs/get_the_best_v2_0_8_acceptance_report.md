# V2-0.8 设施放置手感与占用反馈基线验收报告

## 范围

本轮继续只处理 Godot 前端交互底座，不接入经营规则，不做设施升级数值。

目标是让设施作为办公室里的可操作对象存在，而不是通过旧式面板完成放置和管理。

## 已完成

- 设施放置预览支持读取设施定义里的 footprint。
- 设施放置合法性继续使用绿色预览，非法位置使用红色预览。
- 设施非法放置会返回更具体的原因：
  - 格子已占用
  - 需要对应房间类型
- 放置成功后会刷新设施渲染，并直接高亮新放置的设施。
- 放置成功后不退出设施工具，方便连续摆放同类设施。
- 单格删除命中设施时，只出售该设施，不再继续删除脚下房间格。
- 框选删除仍保留“出售设施并删除房间格”的批量编辑逻辑。

## 设计边界

- 办公桌仍要求研发室。
- 产品白板仍要求市场室。
- 服务器机柜仍要求服务器室。
- 本轮不处理设施移动、升级、折旧、维护费和经营产出。
- 本轮不添加员工，不改变 C# Core 规则边界。

## 本地验收

- 脚手架测试增加 V2-0.8 设施交互检查。
- Godot C# 工程可正常构建。
- 后续仍需要在真实窗口中人工确认 HUD 鼠标点击，因为当前 MCP 鼠标注入对 Control UI 点击存在不稳定边界。

## 下一步建议

进入 V2-0.9：员工占位、选中与批量选择入口。

重点是让员工成为办公室里的可见对象，并复用本轮设施选中、高亮、指针提示和批量选择的交互基础。
