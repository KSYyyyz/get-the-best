# V2-1.6 经营日历与暂停编辑验收报告

日期：2026-05-23

## 本轮目标

把时间流速从“测试倍率按钮”推进到第一局闭环需要的经营时间轴基线，并修复暂停时间后影响玩家编辑操作的问题。

## 已完成内容

1. 经营日历 HUD
   - 新增 `BusinessCalendarHudController`，显示当前经营月和经营日。
   - 暂定每月 20 个经营日，先服务第一局闭环和月报触发验证。
   - 月末进入“月报待查看”状态，并等待玩家恢复时间后继续推进。

2. Core 月末触发桥接
   - `EmployeeAutonomyController` 在 Core tick 时推进经营日历。
   - 月末 tick 会把 `isMonthEnd` 传入 `V2CoreBridge`。
   - `V2CoreBridge` 通过 `SimulationTickOptions.IsMonthEnd` 通知 C# Core。
   - 收到 `MonthlyReportReady` 事件后，前端自动暂停时间，给后续月报面板留出入口。

3. 暂停不阻塞玩家编辑
   - 员工拖拽预览和手动放置动画改用编辑 Tween。
   - 自主行走、工作循环动画继续使用仿真时间缩放。
   - 暂停时间只冻结模拟推进，不冻结设施放置、员工拖拽、预览刷新等玩家操作。

## 验收重点

- `Space` 暂停后，玩家仍然可以移动员工、查看预览并放下。
- 月末触发后，HUD 显示月报待查看，时间自动暂停。
- 玩家恢复时间后，经营日历继续推进。
- Godot 仍只负责表现层时间、HUD 和交互触发，月末经营结算仍由 C# Core 决定。

## 后续限制

当前月报只完成触发与自动暂停基线，还没有正式月报弹窗、月报条目列表、失败/胜利复盘选择入口。下一阶段应把 `MonthlyReportReady` 事件接到真正的月报/复盘面板。
