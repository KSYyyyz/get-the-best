# V2-0.1 办公室沙盒与镜头基线验收报告

状态：已完成
日期：2026-05-20
适用项目：《壮志凌云》/ Get The Best

## 1. 本轮目标

V2-0.1 的目标是把 V2-0 的空办公室骨架收口为可试玩的沙盒地图和镜头基础体验。

本轮只处理办公室空间、窗口适配、镜头控制和基础点击反馈，不进入建造预览、房间创建、设施摆放、员工行为或经营规则。

## 2. 已完成内容

- 办公室区域从单屏占位扩展为 3200x2000 的沙盒地图。
- 主场景显示办公室地板和可读网格。
- Godot 窗口默认最大化启动，并允许玩家缩放窗口。
- `1280x720` 仅作为基准逻辑尺寸，不作为可玩世界尺寸。
- HUD 会根据当前窗口尺寸重新布局。
- 相机支持 WASD 平移。
- 相机支持右键或中键拖拽移动。
- 鼠标滚轮支持大范围缩放，缩放范围为 0.25 - 3.25。
- 滚轮缩放围绕鼠标所在位置调整相机，减少地图缩放时的漂移感。
- 左键点击办公室沙盒区域会更新上下文面板。
- Godot MCP 已接入新项目，并可用于真实运行、截图、点击、拖拽、滚轮缩放和错误检查。

## 3. 本轮明确不做

V2-0.2 不在本轮范围。

以下内容留到 V2-0.2 或后续阶段：

- 鼠标 hover 格子高亮。
- 当前格子坐标常驻显示。
- 拖拽框选区域预览。
- 绿色或红色建造合法性反馈。
- 右键或 Esc 取消建造模式。
- `InteractionRoot` 分层。
- `BuildModeController` 和 `PlacementPreviewController` 雏形。
- 房间真实创建。
- 设施摆放。
- 员工行为。
- 月结、收入、用户、现金流可支撑时间等经营规则。

## 4. 验收依据

本轮验收以真实 Godot 运行和本地自动化检查为准：

- Godot 主场景能在新项目中运行。
- MCP 截图显示大地图网格，而不是单屏面板或小占位框。
- MCP 点击办公室区域后上下文面板会更新。
- MCP 右键拖拽后相机位置发生变化。
- MCP 连续滚轮缩小后相机缩放能到 0.25。
- MCP 连续滚轮放大后相机缩放能到 3.25。
- `get_errors` 返回 0 error。
- 文档检查、脚手架测试、Godot C# 构建和 Godot headless import 均通过。

## 5. 下一步

下一步进入 V2-0.2：办公室格子交互与建造预览基线。

V2-0.2 的重点是把沙盒地图变成可操作棋盘，而不是接入经营规则。只有当 hover、点击、框选、取消、预览和交互分层完成后，才进入 V2-1 第一局经营闭环。
