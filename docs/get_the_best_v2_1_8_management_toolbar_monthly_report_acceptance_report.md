# V2-1.8 经营工具栏与月报窗口验收报告

日期：2026-05-23

## 本轮目标

把常驻 HUD 从调试信息面板改成经营模拟游戏的底部工具栏，并把月报从文字浮层升级为真正的经营数据窗口。

参考方向来自《疯狂游戏大亨》系列的底部公司状态栏、功能入口菜单和居中数据窗口：常驻层只承担快速读数与入口，详细经营数据进入专门窗口。

## 已完成内容

1. 底部公司工具栏
   - 常驻区域从左到右保留公司 Logo、公司名、现金和用户数。
   - 隐藏原本常驻展示的 MVP、MRR、阶段、最近事件、目标和复盘文本。
   - 现金和用户数仍然来自 C# Core 的 `CoreOfficeSimulationResult`，Godot 只做展示。

2. 底部功能入口
   - 建造、购买设施、员工管理、管理部、发行部、统计数据改为同一底部工具栏入口。
   - 建造和购买设施继续调用既有建房/放设施逻辑。
   - 员工管理、管理部、发行部、统计数据先提供可见菜单入口，为后续规则和窗口接入预留位置。

3. 月报窗口
   - 月报改为白底、边框、标题区、分组数据行的居中窗口。
   - 月报内展示现金结余、开发进度、用户数量、本月收入和经营说明。
   - 月报仍由 `MonthlyReportReady` 触发，不在 Godot UI 内复制经营规则。

## 验收重点

- 常驻 HUD 不再直接显示 MVP、MRR、阶段和复盘调试文本。
- 底部工具栏包含公司身份、现金、用户数和六类经营入口。
- 点击建造/设施/员工/管理部/发行部/统计时能看到对应二级菜单。
- 月末触发后出现居中月报窗口，窗口有明确标题、数据行和继续经营按钮。
- `get_errors` 必须为 0。

## 后续限制

当前员工管理、管理部、发行部、统计数据仅完成入口和菜单，不执行经营规则。后续应由 C# Core 提供对应查询、发行、融资、统计等结果，Godot 再接入窗口展示。
