# 《壮志凌云 / Get The Best》V2-0.10 3D 模型化基线验收报告

状态：已完成
日期：2026-05-21
范围：Godot 桌面前端，房间建筑与设施表现层

## 本轮目标

本轮继续只处理 Godot 表现层，不接入经营规则。

目标是把当前“色块 + 贴图牌”的临时表现改成可适配后续 3D / 2.5D 美术资源的模型化基线：

- 设施不再依赖 `Sprite3D` 贴图牌。
- 办公桌、产品白板、服务器机柜使用过程化 3D 模型组合。
- 房间边界从低矮描边升级为半高墙体。
- 房间门从地面短条升级为带门框的入口。
- 办公室外边界墙同步提高到半高墙体。

## 已完成

- `Facility3DRenderer` 改为过程化 3D 设施模型：
  - 办公桌：桌面、桌腿、显示器、椅子。
  - 产品白板：白板面、框架、托盘、便签。
  - 服务器机柜：机柜主体、面板、状态灯。
- `RoomOverlay3DRenderer` 改为建筑化房间表现：
  - 每个房间格仍保留地面铺色。
  - 房间外边界使用半高墙体。
  - 门洞边缘生成门框。
  - 选中房间时墙体使用高亮色。
- `OfficeBoundary3DRenderer` 将办公室外边界墙提高到与房间墙一致的尺度。

## 设计边界

- 本轮不是最终美术，不新增第三方模型资产。
- 过程化模型只作为尺寸、遮挡、选中和后续 glTF/FBX 替换的基线。
- 设施仍然是 1x1 建造格占地。
- 不在 Godot 表现层计算经营产出。

## 验收要求

- 脚手架测试确认设施不再使用 `Sprite3D`。
- 脚手架测试确认房间使用半高墙与门框。
- Godot C# 工程可正常构建。
- Godot MCP 真实运行主场景，错误面板为 0 error。

## 验收结果

- `tests/test_godot_v2_scaffold.py` 新增 V2-0.10 3D 模型化检查。
- Godot MCP 真实运行 `res://scenes/main.tscn`，办公室外边界半高墙可见。
- Godot 错误面板：0 error。
- 本地 `dotnet build`、`pytest`、文档检查和 Godot headless import 均通过。

## 保留问题

- 本轮过程化模型仍是占位表现，不是最终美术。
- 后续如果接入 glTF/FBX 资产，应复用当前 1x1 格占地、墙高、门框和设施锚点，不再回到贴图牌。
- MCP 鼠标注入对“建房后选择门并确认”的长链路仍不够稳定，后续可补一个专用模型预览场景用于快速目测房间和设施。
