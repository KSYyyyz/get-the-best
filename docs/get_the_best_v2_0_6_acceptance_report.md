# 《壮志凌云 / Get The Best》V2-0.6 2.5D 办公室深度提示验收报告

状态：已完成
日期：2026-05-20

## 本轮目标

本轮不扩展经营规则，只修正 V2-0.5 之后“技术上已经是 3D，但视觉仍像平面网格”的问题。

目标是给办公室主场景补上第一层 2.5D 深度提示：

- 办公室边界不再只靠网格线表达，而是有低矮墙体和顶沿。
- 房间不再只是半透明平面，而是拆成地毯、低矮边框和门牌占位。
- 设施不再只依赖 `Sprite3D` 贴图，而是先有 3D 体块，再叠加临时贴图。

## 已完成内容

1. 新增 `OfficeBoundary3DRenderer`
   - 在办公室世界边缘生成四面低矮墙。
   - 使用独立墙体材质和顶沿材质，形成更明确的空间边界。
   - 节点挂载在 `main.tscn` 的 `OfficeWorld/OfficeBoundary3D`。

2. 强化房间覆盖层
   - 房间覆盖层改为地毯层。
   - 每个房间生成低矮边框。
   - 每个房间生成门牌占位块，为后续房间名牌和图标预留位置。
   - 选中房间时仍通过边框高亮表达，不恢复每格黄色高亮。

3. 强化设施 3D 占位
   - 办公桌、产品白板、服务器机柜均先生成 3D 体块。
   - 产品白板和服务器机柜额外生成竖向面板。
   - 临时 Kenney 贴图继续保留，作为 3D 体块上的占位视觉。

4. 增加 scaffold 测试
   - 新增 V2-0.6 测试，锁定办公室边界、房间 2.5D 层和设施体块语义。

## MCP 实机观察

本轮通过 Godot MCP bridge 运行了 `res://scenes/main.tscn`：

- 场景可启动。
- `OfficeBoundary3D` 已出现在 Godot 场景树中。
- Godot 错误面板为 0 error。
- 截图可见底部办公室边界墙。

同时观察到一个下一轮要处理的问题：

- 当前相机仍把大地图铺得太满，边界墙只有底部明显，左右和上边界不够进入玩家第一眼视野。
- MCP 远程点击可以切换菜单，但这次远程拖拽没有成功落地房间；后续仍应继续以真实游玩验证为准，而不是只看静态截图。
