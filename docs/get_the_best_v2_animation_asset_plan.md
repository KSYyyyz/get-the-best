# Get The Best 动画与模型素材准备计划

日期：2026-05-22

## 目标

《壮志凌云》的办公室需要逐步从“可拖拽模型”升级为“可读、可操作、可模拟的经营空间”。素材准备要服务员工自主行动系统，而不是只替换外观。

需要提前覆盖的动作和资源：

- 员工行走、待机、坐下、站起、坐姿工作、打字、交谈、休息、使用设施。
- 门的开门、关门、门框贴墙、房间出入口识别。
- 办公桌、椅子、电脑、白板、服务器、休息设施、卫生间、会议桌等带交互锚点的设施。
- 统一骨架或可复用动作库，避免每次新增动作都重新适配。

## 候选素材源

### Kenney

- 地址：https://kenney.nl/assets
- 重点资源：https://kenney.nl/assets/animated-characters-protagonists
- 许可：Creative Commons CC0。
- 适用：当前低多边形风格、快速原型、可直接进入仓库。
- 风险：动作数量偏少，办公坐姿和打字动作需要补充或自制。

### KayKit

- 地址：https://opengameart.org/content/kaykit-character-animations
- 许可：CC0，页面标注可个人和商业使用、无需署名。
- 适用：Idle、Walk、Run、Interact 等基础动作库，适合作为 V2 员工行走与交互动作基线。
- 风险：不一定直接包含办公坐姿打字，需要和角色包、Blender 处理流程一起验证。

### Quaternius

- 地址：https://quaternius.com/packs/ultimatedanimatedcharacter.html
- 许可：页面标注 CC0，个人和商业项目可用。
- 适用：大量低多边形动画角色，适合做员工类型扩展和备用角色库。
- 风险：格式偏 FBX/OBJ/Blend，需要验证 Godot 4 导入链路，优先通过 Blender 输出 GLB。

### Mixamo

- 地址：https://www.mixamo.com/
- 适用：坐姿、打字、走路、开门、交互等动作原型非常丰富，适合验证动作方向。
- 风险：需要 Adobe 账号；不可把角色和动画作为独立素材再分发。若进入仓库，必须确认许可和再分发边界，优先作为内部原型参考或由我们自制等价动作。

## 推荐落地路线

### A. 当前短期

- 保留现有 Kenney 风格低多边形角色。
- 在 Godot 表现层用座位锚点、面向电脑、缩放、前倾和轻微打字摆动实现工作可读性。
- 设施继续暴露交互点：办公桌座位点、白板前站位、服务器前站位。

### B. 下一阶段

- 选择一个 CC0 角色与动作包作为统一骨架基线，优先验证 KayKit 或 Quaternius。
- 建立 `res://assets/characters/` 与 `res://assets/animations/` 目录规范。
- 为每类设施定义交互锚点：
  - `seat_anchor`：坐下工作。
  - `stand_anchor`：站立操作。
  - `look_at_anchor`：朝向电脑、白板、服务器或门。
  - `door_hinge_anchor`：门开合轴心。

### C. 中期目标

- 员工自主行动从“移动到格子”升级为“移动到设施锚点并播放动作”。
- 办公桌：坐下、打字、短暂停顿。
- 白板：站立书写或讲解。
- 服务器：站立操作、检查灯光。
- 门：靠近后开门，经过后关门。

## 资产入库要求

每次新增素材必须同时记录：

- 来源网址。
- 许可类型和许可页面。
- 是否允许商业使用。
- 是否允许随游戏仓库分发。
- 原始文件、导入文件、Godot 场景封装和使用位置。
- 如果来自 Mixamo，必须额外记录账号下载来源和不可独立再分发限制。

## 当前结论

短期不应为了一个坐姿效果仓促导入许可不清或骨架不统一的角色包。本轮先把“工作位吸附”和“工作姿态锚点”做成可替换的前端结构，下一阶段再以 CC0 动画角色包替换员工模型并接入真实骨骼动画。
